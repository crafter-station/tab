# Support

Use the issue tracker for project support that benefits from public discussion.

## Before Opening An Issue

- Search existing issues and pull requests.
- Check the [README](./README.md), [Contributing Guide](./CONTRIBUTING.md), [PRD](./docs/PRD.md), and [domain glossary](./CONTEXT.md).
- Remove secrets, tokens, prompts, generated text, raw typing logs, and private user data from screenshots or logs.

## Where To Ask

- Bugs: use the bug report issue template.
- Feature ideas: use the feature request issue template.
- Docs fixes: use the documentation issue template.
- Setup or contribution help: use the contributor help issue template.
- Security or privacy vulnerabilities: follow [SECURITY.md](./SECURITY.md), not a public issue.

## Helpful Details

For desktop issues, include macOS version, hardware, target app, permissions state, and whether the issue reproduces after restarting Tab. For API or web issues, include the command, route, environment, and redacted logs.
