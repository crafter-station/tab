# Sidebar Label Motion Plan

## Goal

Make the dashboard logo wordmark and sidebar item labels disappear before their containers become visibly narrow, while preserving a polished reveal on expansion and keeping every icon stationary.

## Implementation

1. Update `apps/web/src/styles.css` only.
   - Keep `.dashboard-sidebar-label` limited to GPU-friendly `opacity` and `transform` transitions.
   - Use the expanded-state transition as the reveal: `opacity 140ms var(--tab-ease-out) 25ms` and `transform 160ms var(--tab-ease-out) 25ms`.
   - Override the collapsed destination selectors for a fast exit: `opacity 60ms linear` and `transform 80ms var(--tab-ease-out)`, with no delay.
   - Preserve the existing 4px horizontal transform; do not animate icon position, padding, control dimensions, or label width.
   - Change keyboard no-motion and reduced-motion overrides from duration-only resets to `transition: none` so the expansion delay cannot leak into those modes.

2. Preserve existing geometry and rendering behavior.
   - Do not modify `apps/web/src/components/pages/dashboard.tsx`; its absolute logo wordmark, single-line labels, clipping, and fixed collapsed trigger widths already prevent reflow and icon drift.
   - Do not modify `packages/ui/src/components/ui/sidebar.tsx`; the motion correction should remain dashboard-scoped and avoid changing shared sidebar behavior.
   - Keep the static sidebar’s existing 160ms structural width transition and its keyboard-triggered no-motion path in `apps/web/src/render-page.tsx`.

## Motion Review Criteria

- Collapse text is effectively gone within 60ms, before clipping becomes visually prominent.
- Expansion waits 25ms for space to open, then reveals with the product’s strong ease-out curve.
- Logo and navigation icons do not translate, resize, or recenter.
- Labels never wrap or appear as compressed multiline text.
- Rapid toggles remain interruptible because the implementation uses CSS transitions rather than keyframes.
- `prefers-reduced-motion` and static `Cmd/Ctrl+B` changes remain immediate.

## Verification

1. Run `bun run --cwd apps/web styles:build`.
2. Run `bun run --cwd apps/web build`.
3. Run `bun run typecheck`.
4. Run `bun test tests/web-account.test.ts`.
5. Run `bun run test`.
6. Manually check collapse and expansion on both static authenticated and hydrated dashboard paths.
7. Confirm click collapse removes text before the sidebar narrows, expansion reveals text only after space opens, and all icons remain stationary.
8. Confirm static `Cmd/Ctrl+B` and `prefers-reduced-motion: reduce` perform an immediate state change without delayed labels.
