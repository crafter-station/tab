# Plan: Maca-Style Desktop Overlay for Tabb

## Goal

Make Tabb's desktop overlay feel like Maca's minimal Crafter Station desktop UI: a small, frosted, animated, bottom-center overlay built with React, while preserving Tabb's existing typing context, suggestion, permissions, tray, preload IPC, and native macOS helper behavior.

## Key Findings

- `tabbb` desktop currently uses raw HTML/CSS/JS in `apps/desktop/src/index.html` loaded by Electron from `apps/desktop/src/main.ts`.
- `tabbb` already has the important desktop behavior: transparent always-on-top non-focusable window, `showInactive()`, suggestion IPC, debug IPC, Option+Tab accept flow, and macOS Accessibility/Input Monitoring onboarding.
- `maca` desktop uses React 19 + Electron Vite, not shadcn/Tailwind. Its overlay design is handcrafted CSS with semantic tokens, frosted black glass, tiny pill states, and CSS animations.
- The most useful Maca code references are:
  - `/Users/cuevaio/.local/share/opencode/repos/github.com/crafter-station/maca/apps/desktop/src/main/windows.ts`
  - `/Users/cuevaio/.local/share/opencode/repos/github.com/crafter-station/maca/apps/desktop/src/renderer/src/components/FloatingBar.tsx`
  - `/Users/cuevaio/.local/share/opencode/repos/github.com/crafter-station/maca/apps/desktop/src/renderer/src/App.css`

## Recommended Approach

Implement a React renderer for the desktop overlay first, not a full desktop app rewrite. Keep `main.ts`, `preload.ts`, settings, onboarding, tray, and native helper behavior mostly intact.

Use shadcn-style utilities/components where they are useful for future settings UI consistency, but do not force Tailwind/shadcn into the overlay animation itself. Maca's overlay look comes from precise CSS, transparent roots, pointer-event control, and Electron window behavior.

## Implementation Steps

1. Add the renderer stack to `apps/desktop/package.json`.
   - Runtime dependencies: `react`, `react-dom`, `clsx`, `tailwind-merge`, `class-variance-authority`, `lucide-react`.
   - Dev dependencies: `electron-vite`, `vite`, `@vitejs/plugin-react`, `@types/react`, `@types/react-dom`.
   - Keep existing Bun-oriented scripts and Electron Builder setup.

2. Add `electron.vite.config.ts` for the renderer entry.
   - Configure only renderer bundling initially.
   - Keep main/preload built by the existing Bun build flow to minimize risk.
   - Output renderer assets under a stable path such as `dist/renderer`.

3. Replace `src/index.html` overlay behavior with a React renderer.
   - Preserve the existing preload API exactly:
     - `window.tabb.onSuggestion(callback)`
     - `window.tabb.onDebugContext(callback)`
     - `window.tabb.onHide(callback)`
     - `window.tabb.acceptSuggestion()`
   - React state model:
     - `mode: "hidden" | "suggestion" | "debug"`
     - `suggestion: { id: string; text: string } | null`
     - `debugContext` matching the current debug payload from `preload.ts`.
   - Register IPC listeners once from `App.tsx`.
   - Clicking the suggestion pill calls `window.tabb.acceptSuggestion()`.

4. Port the Maca visual language, adapted to Tabb.
   - Use transparent `html`, `body`, and `#root`.
   - Set `pointer-events: none` globally; enable `pointer-events: auto` only on visible overlay surfaces.
   - Use a compact frosted black suggestion pill/card with:
     - near-black glass background
     - 24px blur
     - 1px soft border
     - subtle amber/Crafter Station accent via `--signal`
     - spring-like CSS transitions
     - short enter/exit animation
   - Keep debug mode visually secondary: a small dark glass card with captured input, API state, and metadata.
   - Avoid copying Maca's microphone/audio bars directly because Tabb is text suggestion UI, not recording UI. Reuse the animation philosophy, not the dictation-specific controls.

5. Harden the overlay Electron window using Maca's window behavior.
   - In `apps/desktop/src/main.ts`, update `createOverlayWindow()` and `createDebugOverlayWindow()` with Maca-like options where supported:
     - `backgroundColor: "#00000000"`
     - `resizable: false`
     - `fullscreenable: false`
     - `hiddenInMissionControl: true`
     - `type: process.platform === "darwin" ? "panel" : undefined`
   - On macOS call:
     - `setVisibleOnAllWorkspaces(true, { visibleOnFullScreen: true, skipTransformProcessType: true })`
     - `setAlwaysOnTop(true, "screen-saver")`
   - Keep:
     - `focusable: false`
     - `transparent: true`
     - `skipTaskbar: true`
     - `showInactive()`

6. Improve overlay positioning.
   - Replace primary-display-only positioning with active-display positioning using:
     - `screen.getCursorScreenPoint()`
     - `screen.getDisplayNearestPoint(cursor)`
     - `display.workArea.x/y/width/height`
   - Add a small helper for bottom-center bounds.
   - Add display recovery for `display-added`, `display-removed`, and `display-metrics-changed`.
   - Keep debug overlay top-center, but also account for display `workArea.x/y`.

7. Add transparent-region click-through.
   - Default overlay to `setIgnoreMouseEvents(true, { forward: true })`.
   - Poll cursor position while the overlay exists.
   - If cursor is inside the window bounds, use `webContents.capturePage({ x, y, width: 1, height: 1 })`.
   - If pixel alpha is greater than zero, allow mouse events so the pill can be clicked.
   - If pixel alpha is zero or capture fails, pass clicks through.
   - Ensure the suggestion pill remains clickable and still sends `accept-suggestion`.

8. Update desktop dev/build scripts.
   - Add `build:renderer` to `apps/desktop/package.json`.
   - Update `build` to include renderer build before copy/package.
   - Update `build:copy` so it copies settings/onboarding and assets, but no longer treats the old `src/index.html` as the primary overlay source.
   - Update `apps/desktop/scripts/dev.ts` to build/copy the renderer into `.dev/src/renderer` or set a `TABB_RENDERER_PATH`/`TABB_RENDERER_URL` env var.
   - Keep existing native helper build and preload path env behavior.

9. Update `main.ts` renderer loading.
   - Load the React renderer HTML for both suggestion and debug overlay windows.
   - Prefer a helper like `getRendererHtmlPath()` so dev and packaged paths are explicit.
   - Keep settings and onboarding on their current HTML files for this pass unless the user later wants those migrated too.

## Verification

Run from `/Users/cuevaio/projects/tabbb`:

```sh

bun run typecheck
bun run desktop:dev
```

Run from `/Users/cuevaio/projects/tabbb/apps/desktop`:

```sh

bun run dev:permissions
```

Manual checks:

- Overlay appears bottom-center on the active display.
- Overlay does not focus or steal typing from the current app.
- Option+Tab still accepts and pastes the current suggestion.
- Clicking the visible suggestion pill accepts the suggestion.
- Transparent overlay regions pass clicks through to underlying apps.
- Debug overlay still receives and renders `debug-context`.
- `hide` clears the React renderer state and hides the window.
- Moving between displays or changing display layout repositions the overlay correctly.
- Existing onboarding/settings/tray behavior still works.

## Non-Goals for This Pass

- Do not rewrite settings and onboarding in React yet.
- Do not introduce a full Tailwind design system unless needed for shadcn primitives.
- Do not change the suggestion loop, memory/client logic, auth flow, or native input tap helper.
- Do not copy Maca's microphone recording controls; adapt its minimal overlay language to Tabb's text suggestion use case.
