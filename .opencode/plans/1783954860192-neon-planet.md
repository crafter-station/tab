# Fix desktop renderer Node-module bundling

## Root cause

`apps/desktop/src/renderer/src/surfaces/SettingsSurface.tsx` imported the runtime value `SUPPORTED_LOCAL_MODELS` from the Node-only main-process module `apps/desktop/src/main/local-inference-prototype.ts`. That placed `node:child_process`, `node:fs`, `node:util`, and other Node dependencies in Vite's browser graph, where Rollup failed on `promisify` from `__vite-browser-external`.

The `main and preload config is missing` warning is expected: this project intentionally builds main and preload with Bun and configures electron-vite only for the renderer. Do not add electron-vite main/preload config or browser polyfills.

The current worktree already includes the essential fix in `SettingsSurface.tsx`: the `LocalInferenceStatus` import is type-only and the UI no longer reads the model catalog at runtime. Preserve that existing user change.

## Implementation

1. In `apps/desktop/src/renderer/src/surfaces/SettingsSurface.tsx`, confirm the main-process import remains `import type { LocalInferenceStatus } ...` and that the generic local-model status copy does not reintroduce `SUPPORTED_LOCAL_MODELS`.
2. In `apps/desktop/src/main/local-inference-prototype.ts`, remove the now-unused `SUPPORTED_LOCAL_MODELS` export. Keep `QWEN_25_3B_Q4_K_M`, which remains the runtime's default model configuration.
3. In `.github/workflows/ci.yml`, add `bun run --cwd apps/desktop build:renderer` as a named desktop renderer build step. This validates the actual Vite module graph and prevents a future renderer-to-main value import from passing typecheck and unit tests unnoticed.
4. Do not change `apps/desktop/electron.vite.config.ts`, `apps/desktop/scripts/dev.ts`, package dependencies, or the lockfile; the hybrid build configuration is intentional and is not the source of the failure.

## Verification

Run from the repository root:

1. `bun run --cwd apps/desktop build:renderer` to reproduce the exact previously failing bundle boundary. The config warning may remain, but the build must complete.
2. `bun run typecheck` to validate imports and removed exports.
3. `bun run lint` and `bun run test` for repository regression coverage.
4. `bun --env-file=.dev.vars apps/desktop/scripts/dev.ts --build-only` on macOS to verify the complete desktop development build without launching Electron.
5. Optionally launch `bun run desktop:dev` and confirm the settings window opens; the warning alone is not a failure.
