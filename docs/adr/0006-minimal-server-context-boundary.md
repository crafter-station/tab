# Minimal Server Context Boundary

Tab's MVP sends only recent typing context for Deep Complete or the bounded inputs and non-text metadata required for Rewrite. A Rewrite request body is limited to request and device identifiers (`requestId`, `deviceId`); Rewrite `mode`; selected text and bounded text before and after it (`selectedText`, `textBeforeSelection`, `textAfterSelection`); the selected range (`selectedRange` with `location` and `length`); target identity (`activeApplication` with `bundleId`, optional `name`, and required `windowId`, plus `focusedElementId` and `textElementId`); context identity, source, and hash (`contextIdentity`, `contextSource`, `contextHash`); redaction summary metadata (`redaction` with `applied`, `redactionCount`, and `kinds`); eligible personalization (`memoryEnabled` and eligible `customWritingInstructions`); and `clientMetadata` (`appVersion`, `platform`). The API fetches eligible Personal Memory server-side rather than receiving memory contents from the desktop. Rewrite text is sent only after local sensitive-data screening; a secure field or any detected secret-like selected or surrounding text suppresses the request instead of transmitting redacted placeholders. Tab does not send raw key events, screenshots, full document or field contents, clipboard contents, browser URLs, window titles, or accessibility tree dumps, preserving a clear privacy boundary around what the native app may transmit.
