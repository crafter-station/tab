# Conditional Rewrite For Reliable Selections

Double-tapping Option invokes Deep Complete at a reliably observed caret and Rewrite for a reliable non-empty selection. Rewrite uses the same cloud entitlement and one-Suggestion response shape, but it is a distinct request mode because its prompt contract, bounded selected-text boundary, output length, stale-target checks, and replacement semantics differ from continuation.

Rewrite selections and returned replacements are limited to 2,000 characters. Rewrite preserves meaning and language, returns plain replacement text only, and consumes one Deep Complete allowance when the API returns a replacement, even if the target becomes stale afterward and Tab refuses insertion. Invalid, suppressed, or oversized selections dispatch no request, and empty or failed responses consume no allowance. Tab suppresses secure or secret-like selected and surrounding text, does nothing when selection state is unknown, never truncates oversized selections, and invalidates a result whenever the app/window, focused or text element identity, caret or selection range, selected text, or bounded surrounding context changes. These constraints trade broader app support and permissive recovery for protection against transmitting sensitive text or replacing the wrong passage.
