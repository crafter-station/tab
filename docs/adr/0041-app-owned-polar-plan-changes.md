# App-Owned Polar Plan Changes

Tab owns paid-to-paid plan changes in the app instead of sending active paid subscribers through a new checkout, because Polar checkout is for starting subscriptions and rejects customers who already have an active paid subscription. Free-to-paid users continue through checkout, paid upgrades update the existing Polar subscription immediately with prorated adjustment on the next invoice, and paid downgrades are scheduled for the next billing period so paid quota is not removed mid-cycle. Paid-to-free changes remain in the Polar Customer Portal because moving off a paid plan is a cancellation/billing-management flow. After a successful Polar update, Tab updates its cached entitlement immediately and still treats later Polar webhooks as the reconciliation path.
